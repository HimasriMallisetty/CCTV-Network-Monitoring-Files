What SecureVision Does

👥
Crowd Detection
Uses YOLOv8 object detection to count people in real-time. Triggers an audio alert and logs the event when the crowd exceeds your configured threshold.
⚠️
Weapon Detection
Employs a Haar Cascade classifier trained on firearm silhouettes to identify potential weapons in the camera field and raise an immediate alert.
Live & Recorded Feeds
Works with your laptop webcam for live surveillance, or upload pre-recorded video files to run detection and analyse historical footage.
Audio Alerts
Each detection module plays a distinct audio alert — crowd_alert.mp3 for crowds and weapon_alert.mp3 for firearms, with cooldown timers to prevent alert fatigue.
Event Logging
All detection events are timestamped and saved to a persistent log file. Review the full history from the Event Logs page, and clear logs anytime.
Configurable Settings
Adjust camera feed mode (live/recorded), crowd detection threshold, and storage limits from the Settings page without touching any code.
How It Works
  • Step 1
    Camera Capture
    A background thread continuously captures frames from the webcam or video file and stores them in shared memory.
  • Step 2
    AI Detection
    A separate detection thread reads frames, runs the selected AI model (YOLOv8 / Haar Cascade), and draws annotation boxes.
  • Step 3
    MJPEG Streaming
    Annotated frames are JPEG-encoded and streamed to your browser as a multipart HTTP response — no plugins or WebRTC needed.
  • Step 4
    Alert & Log
    When thresholds are breached, a SocketIO event triggers the browser alert feed, an audio file plays, and the event is written to the log.
